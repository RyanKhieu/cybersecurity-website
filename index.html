Data States

Data at Rest

Data is in storage ("at rest"). Examples include files and databases that are stored on hard drives, USB drives, memory, DVDs, and local or remote servers.

Data in Transit

Data is in transmission ("in transit"). The data is moving between systems over the local area network (LAN) or the internet.

Data in Use

Data is being processed. Examples include files being created by a user, data being accessed by an application, or data being processed by memory.

Types of Authentication

What You Know

This type of authentication relies on something the user knows, like a password or PIN. It is the simplest form of authentication but can be vulnerable to attacks like phishing.

What You Have

This type of authentication relies on something the user possesses, like a security token, smart card, or mobile device. It adds an extra layer of security beyond just a password.

What You Are

This type of authentication relies on the user's unique biological traits, like fingerprints, facial recognition, or iris scans. It is considered one of the most secure forms of authentication.

Multifactor Authentication

This type of authentication combines two or more of the other factors to verify a user's identity. This can include a combination of something the user knows (like a password), something the user has (like a token), and something the user is (like a fingerprint).

Passwords

Strong Passwords

Include a mix of uppercase and lowercase letters, numbers, and special characters. It should not be less than 12 characters long and not contain easily guessable information like birthdays or dictionary words.

Lockout

Temporarily disables access after a certain number of failed login attempts. This helps to prevent unauthorized access through guessing.

Multifactor Authentication

Even if passwords are compromised, multifactor authentication can help protect user accounts by requiring additional verification methods.

Hashing

Hashing is a process that transforms a password into a hash value. This hash value is stored instead of the actual password. When a user logs in, the system hashes the entered password and compares it to the stored hash value. If they match, access is granted. The hash value cannot be converted back to the original password, making it more difficult for attackers to obtain users' passwords.

Phishing

Phishing

Phishing is a type of cyber attack where attackers impersonate legitimate organizations or individuals to fool users into revealing sensitive information, such as passwords or credit card numbers. This is often done through fake emails, messages, or websites.

Spear-phishing

A targeted form of phishing where attackers focus on a specific individual, organization, or business, often using personalized information to make their attack more convincing. This can involve spoofing email addresses or creating fake websites that resemble legitimate ones.

Whaling

A specific type of phishing attack that targets high-value individuals, such as executives or important figures within an organization.

Smishing

A type of phishing attack that occurs over text messages. Attackers send fraudulent messages to fool users into revealing personal information or clicking malicious links.

Vishing

A type of phishing attack that occurs over voice calls or messages. Attackers impersonate legitimate organizations or individuals to fool users into revealing sensitive information.

Social Engineering

Social Engineering

Social engineering is manipulation of individuals to get sensitive information or unauthorized access. The attacker uses methods like deception and trickery.

Techniques

Social engineering techniques include baiting, shoulder surfing, pretexting, tailgating, information storage, dumpster diving, pretexting, and scareware. These methods exploit human psychology to trick individuals into giving away confidential information or granting unauthorized access.

Protection

Protection against social engineering attacks include physical security measures, shielding your screen from onlookers, being aware of your surroundings, and using malware scanners.

Potential Threats

Malware

Malware is malicious software designed to compromise a computer system. This includes viruses, worms, trojan horses, ransomware, and spyware.

Social Engineering

Attackers can use social engineering to gain unauthorized access to computer systems by using deception and manipulation tactics on individuals with access.

Man-in-the-middle Attack

A man-in-the-middle (MITM) attack occurs when an attacker intercepts communications between two parties who believe they are communicating directly with each other. This can lead to unauthorized access to sensitive information.

Denial-of-service Attack

A denial-of-service (DoS) attack occurs when an attacker overwhelms a system with traffic, making it unavailable to users.

Confidentiality

Confidentiality is protecting sensitive data from being exposed to unauthorized access. This can be achieved by controlling access to data by only allowing relevant users to access it, encrypting sensitive data, and using multi-factor authentication.

An example of good confidentiality practices is limiting departments in a company only to relevant data that they need for their specific roles.

×

Integrity

Integrity is making sure data can be trusted and cannot be tampered with. This can be achieved by using hashing, digital signatures, encryption, and digital certificates.

An example of good integrity practices is ensuring that all websites accessed by an organization are using HTTPS and have valid SSL certificates.

×

Availability

Availability ensures that data is accessible to relevant users when needed. This can be achieved through redundancy, backups, and regular maintenance and updates.

An example of good availability practices is implementing backup systems and regularly testing backup and recovery procedures.
